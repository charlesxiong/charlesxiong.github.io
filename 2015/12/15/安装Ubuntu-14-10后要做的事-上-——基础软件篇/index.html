由于平时主要在Ubuntu上工作,折腾Ubuntu的时候比较多。这里主要是总结使用Ubuntu的常见问题,分为两部分,第一部分是常见基础软件的使用。

更新软件源

装完ubuntu第一件事就是添加更快速的源,因为官方提供的源服务器在国外,访问和下载速度要比国内源慢些,可以采用一些国内公司和大学提供的更新源.

  • 在System Settings -> Software & Updates,在Download from中在中国区选择速度最快的服务器源。这种方法比使用命令行快很多。

  • 命令行设置步骤:

    1. 备份原有源,恢复时有用 sudo cp /etc/apt/sources.list /etc/apt/sources.list.backup
    2. 修改更新源,添加相应的镜像源(个人常用163源、yun-idc源) sudo gedit /etc/apt/sources.list
    3. 更新源并升级系统

文本乱码

缺省配置下,用Ubuntu的文本编辑器（gedit）打开GB18030（繁体中文用户可将GB18030替换成BIG5或BIG5-HKSCS）类型的中文编码文本文件时会发生乱码。这是因为gedit使用一个编码匹配列表,只有在这个列表中的编码才会进行匹配,不在这个列表中的编码将显示为乱码。您要做的就是将GB18030加入这个匹配列表。

  • Gedit 3.x版本设置（适用于Ubuntu 11.10及以后）

    • 安装dconf-editor: sudo apt-get install dconf-tools
    • 手动方式：在org/gnome/gedit/preferences/encodings下添加中文编码方式

  • Gedit 2.x版本设置（适用于Ubuntu 11.04及以前）

开启自动挂载分区

  1. 在root权限下查看分区: fdisk -l

  2. 统计windows分区,命令: sudo mkdir -p /media/charles/{windows分区名字}

  3. 编辑描述各种文件系统信息的文件/etc/fstab,添加对应Windows分区信息:sudo gedit /etc/fstab

远程登陆主机

默认在Linux上安装ssh服务器： sudo apt-get install openssh-server

命令行管理

  • ubuntu下开启ssh服务
    查看ssh服务是否在后台开启,出现了sshd字样表示ssh后台服务默认已开启

你的服务器也可以是Mac或者Windows,只需要安装ssh服务器,并开启ssh后台服务即可。如果开启ssh服务后客户端连接不上服务器,在服务器上执行如下命令:

  • Windows下安装ssh客户端
    • secure shell client：免费开源,但中文显示乱码,无法传输中文乱码名的文件,不能记住密码
    • putty : 占用内存最小,每次登陆时需要重新连接,可通过设置编码为UTF-8不会显示中文乱码
    • Xshell和Xftp: 商用软件,但有免费版,可以保存设置的登陆会话,能记住密码,传输文件和显示中文时都可以通过设置编码为UFT-8而不会产生乱码,效果最佳。

推荐使用XShell+Xftp软件,创建新会话,填写完ip地址、用户名和密码就可登陆你的Ubuntu服务器
