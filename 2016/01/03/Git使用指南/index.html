创建SSH密钥

要想在你的机器上提交代码到远程服务器的仓库，Github或GitCafe等代码托管网站统一使用SSH key的认证方式保证通信安全。

  1. 检测SSH key

  2. 生成新的SSH key,输入passphrase

  3. 添加SSH公钥到Github里
    用文本编辑器打开公钥文件并复制所有内容或使用clip < ~/.ssh/id_ras.pub，在Github上选择
    Account Settings -> SSH keys ->Add SSH key,把新的公钥信息粘贴进去即可

  4. 测试本机器是否可以连接到Github服务器上

关联本地与远程仓库

在Github上添加一个Git仓库后，想对该仓库进行修改一般有两种手段:

  • 从远程库克隆到本地
  • 本地创建空仓库后将其与远程仓库关联

一般地建议先pull服务器的数据到本地上，完成主要的开发工作后使用push推送更新

管理本地仓库

和SVN不同的是，Git有几个专业术语需要重点解释:

  • 工作区(Working directory, workspace):　实际存在的文件，隐藏目录.git不算工作区
  • 暂存区(Index,Stage): 被Git管理的临时保存内容
  • 本地仓库(Local repository): 指向提交后的各种分支版本，它包括自动创建的master分支、指向Master分支的HEAD指针
  • 远程仓库(Remote repository): 本地仓库的异地备份，有可能与本地不同步，如Github/Gitcafe上存储的就是远程仓库

它的整体过程是:

git add -> git commit -> git push

具体意思是:

在工作区中创建或修改后被添加进暂存区，暂存区的内容开始被版本控制，
之后被提交到本地仓库，最后被推送到远程仓库与多人进行协作

所以对本地仓库的修改无非就是添加、删除、提交、分支管理等这些命令的操作使用。

添加、删除操作
例如在readme.txt中添加了一行信息:

现在有一个问题需要考虑: 我想回退所做的工作，该怎么做。有以下两种情况

  • 回退提交的版本

我已经进行了三次commit到本地仓库里,现在想回退到某个特定的版本，怎么做?
如果回退到某版本后想重返未来又该怎么处理?

使用git log命令，它显示从最近到最远的提交日志。每进行一次提交时Git会将它们自动串成一条时间线，若使用可视化工具查看Git提交历史，时间线会比较清楚的显示出来

回退版本使用git reset --hard命令，它有两种方式:

  1. 使用提交的版本号，只需要前7位即可

  2. 基于HEAD指针的表示。Git仓库内部维护了一个指向当前版本的HEAD指针，所谓的回退实际上是指HEAD指向了其他commit位置，并顺便把工作区的文件更新到那个位置。HEAD表示当前版本，上一个版本就是HEAD^,上上一个版本是HEAD^^,往上50个版本则写成HEAD~100。回顾之前说的，

现在假设已经回到了modify readme版本时，若还想恢复到原来的first cm处，这时它就必须找到对应的commit id。使用git reflog命令即可满足要求，它记录了你的每一次命令，之后再用git reset --hard命令回到未来的某个版本。

  • 回退工作区或暂存区的修改

场景1　修改了工作区的内容但并未添加到暂存区

想直接丢弃工作区的修改，使用命令git checkout -- filename

场景2　工作区的内容修改后还被添加到了暂存区，但并未被提交。

若想放弃修改，使用git reset HEAD file可把暂存区的修改撤销掉(unstage),重新放回工作区，然后再按场景1操作
